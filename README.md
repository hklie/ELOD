# ELOD - ELO Rating System for Duplicate Scrabble Tournaments

**Version 1.1 — March 2026**

ELOD is a rating calculator designed for **Scrabble Duplicate tournaments** organized under the FILE (Federacion Internacional de Lexico en Espanol) framework. It processes tournament results from text files, Microsoft Access databases, and images, then calculates and tracks player ELO ratings over time.

## Overview

ELOD processes tournament results from multiple events over multiple years, comparing each pair of players by their ranking position, and computing ELO rating changes based on experience-adjusted K-factors. It produces per-tournament CSV ranking files and a consolidated Excel workbook that tracks every player's progression.

ELOD ratings corresponding to **Copa FILE** players are automatically calculated from each world duplicate game.

### Key Features

- Calculate ELO ratings from tournament rankings
- Support for `.txt`, `.mdb`, `.accdb`, `.html`/`.htm`, and image (`.jpeg`, `.png`) input formats
- Experience-based K-factor: higher adjustment for newer players
- Player name alias system to unify spelling variations and accents
- Exclusion of deceased players from final rankings
- Progressive tracking: an Excel workbook with per-tournament ELO deltas
- Manifest system for organizing tournaments chronologically
- Auto-initialization of new players with a configurable default rating

## Input Files

All input data lives under the `data/` directory, organized by year.

### 1. Tournament Files

These contain the tournament results — player names ordered by final ranking (1st place first).

| Format | Extension | Description |
|--------|-----------|-------------|
| Text | `.txt` | One player name per line, in ranking order |
| Access DB | `.mdb`, `.accdb` | Microsoft Access database generated by **DupMaster** (the program used to manage Duplicate tournaments), with `Jugadores` and `Relaciones` tables. |
| HTML | `.html`, `.htm` | HTML classification page exported by **DupMaster** (e.g., `Clasificacion.html`). No external dependencies required. |
| Image | `.jpeg`, `.png` | Scanned results (processed via OCR; a matching `.txt` file takes priority) |

**Text file example** (`Europeo_2025.txt`):
```
SergeEmig
AntonioAlvarez
CarlosGarcia
MarinaColson
ArantxaDelgado
```

**Access database tables:**

| Table | Columns | Purpose |
|-------|---------|---------|
| `Jugadores` | `Indice`, `Nombre`, `Acronim` | Player information |
| `Relaciones` | `Jugador`, `Ronda`, `PuntuacionAcumulada` | Scores per round |

**Location:** `data/YYYY/` (e.g., `data/2025/Europeo_2025.txt`)

### 2. Tournament Manifest (`.manifiesto`)

The manifest is the master list that tells ELOD which tournaments to process and in what order. Each tournament entry has a date line followed by one or more tournament files.

**Format:**
```
DD-MM-YYYY  # Event Display Name
tournament_file1.txt
tournament_file2.mdb
```

**Example** (`data/torneos_all.manifiesto`):
```
1-08-2025 # Europeo Murcia 2025
2025/Europeo_2025.txt

01-11-2025  # Mundial Santiago de Chile 2025
2025/Duplicada Mundial 1 - 2025.accdb
2025/Duplicada Mundial 2 - 2025.accdb
2025/Duplicada Mundial 3 - 2025.accdb
```

Each year also has its own manifest in `data/YYYY/torneos_YYYY.manifiesto`. The script `regenerate_all.py` combines them into `data/torneos_all.manifiesto`.

### 3. Player Aliases File (`.alias`)

Maps variant spellings, accents, and name formats to a single canonical name.

**Location:** `data/jugadores_all.alias`

**Format:**
```
# canonical_name = alias1, alias2, ...
AiranPérez = AiranPerez
XavierPiqué = XaviPiqué
JoséGonzález = GONZALEZ,José; GONZALEZ,J
```

### 4. Deceased Players File

Players listed here are excluded from the final rankings (but are still included in ELO calculations).

**Location:** `data/jugadores_fallecidos.txt`

```
FreddyRamírez
EnriqueCortés
MarthaGalindo
```

### 5. Display Names File

Custom display names for players with compound surnames that the automatic algorithm cannot split correctly.

**Location:** `data/nombres_display.txt`

```
DavidLozanoGaray = LOZANO GARAY, David
MaríaMartaGismondi = GISMONDI, María Marta
MaríadeArcos = DE ARCOS, María
```

### 6. Initial Player Ratings File (`.elod`) — Optional

If you want to start from pre-existing ratings instead of auto-initializing everyone, provide a `.elod` file with tab/space-separated columns:

```
PlayerName    ELO    Games    Tourneys    LastTourney
Alice         1800   0        0           -
Bob           1750   50       10          tournament5.txt
```

## How to Add Tournament Results

This section explains the step-by-step workflow for incorporating results from a new tournament.

### Step 1: Prepare the Tournament File

- If you have a **text ranking**, create a `.txt` file with one player name per line in final ranking order (1st place on the first line). Use CamelCase with no spaces (e.g., `CarlosGarcia`, `XavierPiqué`).
- If you have a **Microsoft Access database** (`.mdb` or `.accdb`) generated by **DupMaster**, place it as-is in the year folder. DupMaster is the program used to run Duplicate tournaments and it exports results as Access databases (as well as HTML classification pages).
- If you only have a **photo/scan** of the results (`.jpeg` or `.png`), place the image in the year folder. For best results, also create a `.txt` file with the same base name as a manual transcription (ELOD will prefer the `.txt` over OCR).

### Step 2: Place the File in the Correct Year Folder

Put the file under `data/YYYY/`, where `YYYY` is the year of the tournament. For example:

```
data/2025/Europeo_2025.txt
```

### Step 3: Update the Year Manifest

Open the year's manifest file at `data/YYYY/torneos_YYYY.manifiesto` and add an entry for the new tournament:

```
DD-MM-YYYY  # Event Display Name
filename.txt
```

For multi-part tournaments (e.g., a World Championship with 3 sessions), list all files under the same date line:

```
01-11-2025  # Mundial Santiago de Chile 2025
Duplicada Mundial 1 - 2025.accdb
Duplicada Mundial 2 - 2025.accdb
Duplicada Mundial 3 - 2025.accdb
```

### Step 4: Check for New Player Names

If any player names are new variants of existing players (different accents, different spelling), add an alias entry in `data/jugadores_all.alias`:

```
CanonicalName = NewVariant
```

You can also run the program with the `--check-names` flag to auto-detect potential duplicates.

### Step 5: Regenerate All Results

Run the regeneration script (see [How to Run the Program](#how-to-run-the-program) below).

## How to Run the Program

### Option A: Windows Executable (Recommended for Windows Users)

No installation required — just download and run.

1. Go to the [Releases](../../releases) page and download the latest `elod-windows-vX.X.X.zip`
2. Unzip the file to a folder on your computer (e.g., `C:\ELOD`)
3. Place your tournament files in the `data/YYYY/` folders (e.g., `data/2025/`)
4. Update the year manifest in `data/YYYY/torneos_YYYY.manifiesto`
5. Double-click **`run_elod.bat`** to process all tournaments
6. Results will appear in the `output/` folder

You can also run `elod.exe` directly from the command line:
```cmd
elod.exe --data-path data --output-path output
```

### Option B: Running from Python Source

#### Prerequisites

You need the following installed on your computer:

1. **Python 3.7 or higher** — Download from [python.org](https://www.python.org/downloads/) and install it. During installation on Windows, check the box that says "Add Python to PATH".

2. **mdbtools** (only needed if you process `.mdb` files):
   ```bash
   # On Ubuntu/Debian Linux:
   sudo apt-get install mdbtools

   # On macOS with Homebrew:
   brew install mdbtools
   ```

3. **Python dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

#### Getting the Code

Download or clone the project folder to your computer. Open a terminal (Command Prompt on Windows, Terminal on macOS/Linux) and navigate to the project folder:

```bash
cd path/to/elod
```

### The Easy Way: Regenerate Everything

The simplest way to run the program is to use `regenerate_all.py`, which reads all year manifests, combines them, and produces all output files:

```bash
python regenerate_all.py
```

This will:
1. Read year manifests from `data/2022/`, `data/2023/`, `data/2024/`, `data/2025/`, etc.
2. Generate the combined manifest `data/torneos_all.manifiesto`
3. Process all tournaments in chronological order
4. Produce all output CSV files and the consolidated Excel workbook in the `output/` folder

**Options:**
```bash
# Only regenerate the combined manifest (skip ELO processing):
python regenerate_all.py --manifest-only

# Specify custom paths:
python regenerate_all.py --data-path data --output-path output
```

### Running Individual Components

**Process tournaments from a manifest:**
```bash
python elod.py \
  --manifest data/torneos_all.manifiesto \
  --aliases data/jugadores_all.alias \
  --deceased data/jugadores_fallecidos.txt \
  --output ./output \
  --auto-init
```

**Generate the Excel workbook only:**
```bash
python generate_progressive.py \
  -m data/torneos_all.manifiesto \
  -a data/jugadores_all.alias \
  -b data \
  -d data/jugadores_fallecidos.txt \
  -n data/nombres_display.txt \
  -o output/elod_progresivos.xlsx
```

**Extract rankings from a single Access database:**
```bash
python mdb_reader.py tournament.mdb --format clasificacion --output rankings.txt
```

**Check for potential duplicate player names:**
```bash
python elod.py \
  --manifest data/torneos_all.manifiesto \
  --aliases data/jugadores_all.alias \
  --auto-init --check-names --quiet
```

### Command-Line Arguments Reference

| Argument | Short | Description |
|----------|-------|-------------|
| `--base-path` | `-b` | Base directory for data files |
| `--players-file` | `-p` | Initial player ratings file (`.elod`) |
| `--tournaments` | `-t` | Tournament files to process (`.txt`, `.mdb`, `.accdb`) |
| `--manifest` | `-m` | Tournament manifest file (alternative to `--tournaments`) |
| `--output` | `-o` | Output directory for results |
| `--auto-init` | `-a` | Auto-initialize new players from tournaments |
| `--default-elo` | `-e` | Default ELO for new players (default: 2075) |
| `--aliases` | | Player name aliases file |
| `--deceased` | | Deceased players file |
| `--check-names` | | Detect potential name duplicates |
| `--quiet` | `-q` | Suppress progress output |

### Running Tests

```bash
python test_elod.py
```

## Results and Output

All output files are placed in the `output/` directory.

### Per-Tournament CSV Files

For each tournament processed, a CSV file is generated with the tournament's name (e.g., `Europeo_2025.csv`). Columns:

| Column | Description |
|--------|-------------|
| `Posicion` | Final ranking position |
| `Jugador` | Player name (canonical, with aliases applied) |
| `ELOD Inicial` | ELO rating before this tournament |
| `Delta ELOD` | Rating change (+ or -) |
| `ELOD Final` | Rating after this tournament |
| `N. Oponentes` | Number of opponents faced |
| `N. Partidas` | Total games played (cumulative) |
| `Ultimo Torneo` | Display name of the tournament |

### Final Rankings CSV

The file `elod_final.csv` (produced by the last tournament processed) contains the overall standings after all tournaments. Same column format as above. Deceased players are excluded from this file.

### Progressive Excel Workbook

The file `elod_progresivos.xlsx` is a consolidated Excel workbook that tracks every player's ELO progression across all tournaments:

- **Title header** with FILE institution name and dynamic generation date (in Spanish)
- Column order: Deltas acumulados, Pos., JUGADOR, País, ELOD Actual, Último Torneo, N.Oponentes, N.Partidas, followed by per-tournament delta columns
- **Vertical tournament headers** for compact display of many tournament columns
- Participant count row below the headers
- Color-coded delta cells (green for positive, red for negative)
- Frozen panes (first 3 columns + first 5 rows) for easy navigation

### Output File List (Example)

```
output/
├── Europeo_2025.csv
├── Duplicada Mundial 1 - 2025.accdb.csv
├── Duplicada Mundial 2 - 2025.accdb.csv
├── Duplicada Mundial 3 - 2025.accdb.csv
├── PartidaBaAs1.csv
├── ...                          (one CSV per tournament)
├── elod_final.csv               (overall final rankings)
└── elod_progresivos.xlsx        (consolidated Excel workbook)
```

## ELO Calculation

### Formula

1. **Expected Win Probability:**
   ```
   E = (1 + erf(rating_diff / 400)) / 2
   ```

2. **K-Factor (experience-based):**
   - New players (< 50 games): K = 25
   - Experienced players (>= 50 games): K = 10

3. **Rating Change:**
   ```
   Delta ELO = K x (actual_result - expected_probability)
   ```

### Tournament Processing

In a tournament with N players, each player is compared pairwise against every other player. A higher-ranked player "wins" each pairwise comparison. Total games per player = N - 1.

**Example:** In a 5-player tournament, the 1st-place player wins 4 pairwise matches (against positions 2-5), while the 5th-place player loses 4.

### Configuration Constants

| Constant | Value | Description |
|----------|-------|-------------|
| `K_NEW_PLAYER` | 25 | K-factor for players with < 50 games |
| `K_EXPERIENCED` | 10 | K-factor for players with >= 50 games |
| `GAMES_THRESHOLD` | 50 | Games needed to become "experienced" |
| `RATING_FACTOR` | 400 | Standard ELO rating difference factor |
| `DEFAULT_ELO` | 2075 | Default rating for new players |
